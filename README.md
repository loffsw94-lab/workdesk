# 워크데스크 — 5분 셋업 가이드

코드는 전부 완성되어 있습니다. **아래 4단계 외에는 아무것도 만지지 마세요.**
수정 가능한 파일은 `config.js` 딱 하나입니다.

---

## 1단계. 배포하기 (5분, 필수)

1. github.com 가입 → 새 저장소(repository) 만들기 → **이 폴더 전체를 드래그해서 업로드**
2. vercel.com 에서 GitHub 계정으로 가입 → **Add New → Project** → 방금 만든 저장소 선택 → **Deploy**
3. 끝. `프로젝트명.vercel.app` 주소로 사이트가 열립니다.
4. (선택) 도메인을 샀다면: Vercel → Settings → **Domains** 에 입력하고 안내대로 DNS만 바꾸면 연결됩니다.

## 2단계. 뉴스 자동 수집 켜기 (5분)

1. developers.naver.com → 애플리케이션 등록 → 사용 API **"검색"** 선택 → Client ID / Secret 발급
2. Vercel → 내 프로젝트 → Settings → **Environment Variables** 에 두 줄 입력:
   - `NAVER_ID` = 발급받은 Client ID
   - `NAVER_SECRET` = 발급받은 Client Secret
3. Deployments 탭에서 **Redeploy** 클릭
4. 끝. 뉴스·스포츠·연예 탭이 최신 기사로 자동으로 채워지고 10분마다 갱신됩니다. (코드 수정 없음)

## 3단계. 외주 폼 받기 (3분)

1. formspree.io 무료 가입 → New Form → 발급된 주소 복사 (`https://formspree.io/f/xxxx`)
2. `config.js` 열어서 `FORM_ENDPOINT: ""` 따옴표 안에 붙여넣기 → 저장 → GitHub에 업로드(커밋)
3. 끝. 의뢰가 접수되면 내 이메일로 도착합니다.

## 4단계. 검색 등록 (10분 — 방문자의 시작점)

1. **네이버**: searchadvisor.naver.com → 사이트 등록 → 소유 확인에서 **"HTML 파일 업로드"** 선택
   → 받은 파일을 이 폴더에 넣고 GitHub에 업로드 → 확인 버튼
2. **구글**: search.google.com/search-console → 동일하게 HTML 파일 방식으로 확인
3. 두 곳 모두 사이트맵 제출: **`https://내도메인/api/sitemap`**
   (도메인은 자동 감지되므로 파일을 고칠 필요 없습니다)

---

## 5단계. 방문자 측정 켜기 (5분, 강력 추천)

1. analytics.google.com 에서 무료 GA4 속성 생성 → 측정 ID(`G-XXXXXXXXXX`) 복사
2. `config.js`의 `ANALYTICS_ID`에 붙여넣기 → 끝. 어떤 페이지로 유입되는지 보입니다.

---

## 나중에 (수익화 시점)

- **애드센스**: 승인받은 뒤 `config.js`의 `ADSENSE_CLIENT`에 `ca-pub-…` ID만 붙여넣으면
  사이트의 모든 광고 자리가 실제 광고로 바뀝니다. (이 사이트는 이미 107개 페이지가 있어
  개인정보처리방침의 괄호만 채우면 바로 심사 신청 가능한 규모입니다)
- **환율**: 이미 실시간 자동 연동되어 있습니다. 키도 필요 없습니다.
- **상담 리드**: 부가세·세금계산서·연봉 계산 결과 아래에 "무료 견적 문의" 버튼이 자동으로
  붙습니다(외주 폼으로 연결). 트래픽이 쌓이면 세무·노무 파트너와 제휴해 이 버튼을
  리드 채널로 쓰는 것이 애드센스보다 큰 수익원이 됩니다.

## 폴더 구조 (참고용 — 만질 필요 없음)

```
config.js        ← ★ 유일하게 수정하는 파일
index.html         메인 허브 (계산기 5종 + 콘텐츠 + 외주 폼)
vat/salary/loan/fx/invoice.html   키워드별 메인 랜딩 5개
rates.html         2026년 4대 보험 요율표 (에버그린 콘텐츠)
s/  (23개)         연봉별 실수령액  ─ "연봉 3000 실수령액" 등  ┐
w/  (9개)          월급별 실수령액                             │
v/  (15개)         금액별 부가세    ─ "100만원 부가세" 등      │ 롱테일 100페이지
a/  (22개)         평수 변환        ─ "84㎡ 몇 평" 등          │ (10만 방문의 엔진)
l/  (24개)         대출 조건별 상환표                          │
f/  (7개)          프리랜서 3.3% 실수령                        ┘
privacy.html       개인정보처리방침 (애드센스 심사용)
assets/            디자인·기능 (공용) — 계산 요율은 assets/app.js 상단 RATES에서 관리

api/robots.js      robots.txt 자동 생성 (사이트맵 위치 포함)
vercel.json
```

## 방문 10만을 위한 운영 팁 (기술 작업 아님)

- 이 사이트는 메인 키워드 랜딩 6개 + **롱테일 검색어 100개**(연봉 금액별·부가세 금액별·
  평수·대출 조건별)마다 전용 페이지를 갖고 있고, 각 페이지에는 검색엔진이 읽는
  **미리 계산된 정적 결과표**가 들어 있습니다. 검색 등록(4단계)만 하면 유입 구조는 완성입니다.
- **계산 결과는 링크로 공유됩니다**: 결과 영수증의 "결과 링크 복사"를 누르면 값이 담긴
  주소가 복사되고, 받은 사람이 열면 같은 계산이 재현됩니다. 커뮤니티 공유에 쓰세요.
- 초기 부스터: 네이버 지식iN의 세금·연봉 질문에 답하며 해당 금액의 롱테일 페이지 링크 첨부
  (예: "연봉 4천 실수령액" 질문 → `s/4000.html` 링크), 직장인 커뮤니티에 도구 공유.
- privacy.html의 괄호 부분(이름·이메일·날짜)만 채워두세요. 애드센스 심사에 필요합니다.
